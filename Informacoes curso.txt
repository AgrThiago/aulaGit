Resumo da Aula

Nesta aula, você aprendeu como conectar um repositório local (no seu computador) com um repositório remoto (no GitHub). O processo envolve configurar o Git com suas informações pessoais, adicionar seus arquivos, criar um commit inicial e estabelecer a conexão segura com o GitHub através de uma chave SSH.

Os principais conceitos abordados foram: a diferença entre repositório local e remoto, a importância da autenticação via SSH para segurança, e como sincronizar seus projetos entre o computador e a plataforma GitHub.

Passo a Passo Completo para Novos Projetos

Aqui está um guia prático que você pode usar sempre que iniciar um novo projeto:

1. Criar repositório no GitHub
Acesse github.com, faça login e clique em "New repository". Escolha um nome para seu projeto e clique em "Create repository". Copie a URL SSH que aparece.

2. Inicializar repositório local
Abra o terminal na pasta do seu projeto e execute:
													git init

3. Configurar identidade no Git (primeira vez apenas)
Se for a primeira vez usando Git neste computador, configure seu e-mail e nome:
git config --global user.email "seu.email@example.com"
git config --global user.name "Seu Nome"

Substitua pelos dados da sua conta GitHub.

4. Adicionar arquivos
Para adicionar todos os arquivos do projeto de uma vez:
git add .

5. Criar commit inicial
git commit -m "projeto inicial"

Você pode alterar a mensagem conforme necessário.

6. Renomear branch para main
git branch -M main

7. Conectar repositório local ao remoto
git remote add origin git@github.com:seu-usuario/seu-projeto.git

Substitua pela URL SSH que você copiou do GitHub.

8. Gerar chave SSH (primeira vez apenas)
Se for a primeira vez, o Git pedirá autenticação. Gere uma chave SSH com:
ssh-keygen -t ed25519 -C "seu.email@example.com"

Pressione Enter para aceitar o local padrão e deixe a senha vazia (pressione Enter duas vezes).

9. Adicionar chave SSH no GitHub
Abra a pasta .ssh no seu computador (geralmente em C:\Users\SeuUsuario\.ssh no Windows). Abra o arquivo id_ed25519.pub com um editor de texto, copie todo o conteúdo. No GitHub, vá em Settings > SSH and GPG keys > New SSH key, cole a chave e salve.

10. Sincronizar com GitHub
git push -u origin main

Pronto! Seu projeto está sincronizado. Você pode compartilhar a URL do repositório com outras pessoas.

Dica importante: Os passos 3, 8 e 9 você faz apenas uma vez no seu computador. Nos próximos projetos, comece direto do passo 1 e pule para o passo 4.


# Resumo da Aula: O comando git remote

## 📌 O que é?
O comando `git remote` conecta seu repositório local (no seu computador) com um repositório remoto (no GitHub).

---

## 🔧 Comandos Principais - Passo a Passo

### 1️⃣ **Adicionar um repositório remoto**
```
git remote add apelido url
```
- **apelido**: nome que você escolhe (geralmente "origin")
- **url**: endereço do repositório no GitHub
- **Quando usar**: Na primeira vez que conecta seu projeto ao GitHub

---

### 2️⃣ **Listar repositórios remotos**
```
git remote -v
```
- Mostra todos os repositórios remotos vinculados
- Aparece duplicado (uma linha para fetch, outra para push)
- **Quando usar**: Para verificar se está tudo configurado corretamente

---

### 3️⃣ **Remover um repositório remoto**
```
git remote remove apelido
```
- **Exemplo**: `git remote remove origin`
- **Quando usar**: Quando não precisa mais daquele repositório

---

### 4️⃣ **Alterar a URL de um repositório remoto**
```
git remote set-url apelido nova_url
```
- **Exemplo**: `git remote set-url origin https://github.com/seu-usuario/seu-repositorio.git`
- **Quando usar**: Se copiou a URL errada ou ela foi modificada

---

### 5️⃣ **Renomear um repositório remoto**
```
git remote rename apelido novo_apelido
```
- **Exemplo**: `git remote rename origin novo-origin`
- **Quando usar**: Para mudar o nome do apelido

---

## 💡 Dica importante
O "origin" é o nome padrão para o repositório principal no GitHub, mas você pode usar outros nomes se necessário!

Principais comandos usados:

git pull origin main - Baixa os novos commits do repositório remoto (origin) para a branch main do seu repositório local.

git log - Verifica os commits que agora estão no repositório local, mostrando autor, data e mensagem de cada um.

O fluxo de trabalho completo:

A aula também reforçou o ciclo de trabalho colaborativo que você aprendeu até agora:

git status - Verifica quais arquivos foram modificados
git add - Adiciona as mudanças para o commit
git commit - Registra as mudanças com uma mensagem
git push - Envia os commits para o GitHub
git pull - Baixa os commits que outras pessoas enviaram
